Choosing a Game Engine

Published 19. February 2021

I've for a long time messed around with different engines / frameworks to try and find the one that's right for me, and it hasn't been easy, and to be quite honest, I'm probably not there yet.

This is the game engines / frameworks I've tried from first to most recent:

  • Unity
  • GameMaker Studio 1.4 / 2
  • LOVE2D 11.3
  • Raylib 3.5
  • Godot 3.2
  • DOME 1.5

Cross-Compilation:

For cross-compilation, Godot is without a doubt the best, with easy support for many platforms and relatively small executables.

DOME is also very simple for this purpose, currently supporting Windows, Mac and Linux, and web support is currently in-development.

LOVE2D and Raylib are not great for this purpose since cross-compilation isn't easy at all, but in terms of platform compatibility Raylib seems to run on pretty much anything.

GameMaker is also pretty good for cross-compilation, if it wasn't for the quite so expensive developer licenses, especially for a student.

Unity also seems good for cross-compilation, and also supports web

Options:

  • Unity
  • Godot
  • DOME

Executable Size

Executable size is very important to me. If it wasn't, I'd have made this game with JavaScript using Electron

Unity is OK when it comes to file size, with TIS-100 (similar to Brainstuck) taking up 45 MB on my harddrive. And it's possible to lower this quite a bit

Godot is pretty good for executable size. With a bit more customization it's possible to lower the executable size even more by disabling stuff such as 3D and a lot of libraries that might not be needed. If I remember correctly I got a "Hello World" application from 25 MB -> 18 MB. That might not seem like a lot, but it's a 28% decrease in executable size.

DOME is perfect for this use case only taking up around 1.5 MB.

Options:

Unity, Godot and DOME can all still be considered, but I'm not to fond of the Unity executable size.

  • Unity
  • Godot
  • DOME

Open-Source

This also means a lot to me because this allows me to follow development, contribute and it keeps the source transparent.

Unity isn't open-source.

Godot is open-source and is being developed very actively.

DOME is also open-source and is also being developed actively.

Options:

  • Godot
  • DOME

If you know anything about Godot, you might think that Godot would be a good choice, since there's more support for it, larger variety of platforms supported, very actively developed, easy GUI control system and more. So I started using Godot to work on Brainstuck, and while the GUI system was nice, I found Godot to be way too overkill for a small game such as the one I was developing. In this case I prefer game-engines such as LOVE2D, Raylib and DOME, without the editor but only pure scripting. For me this makes it easier to organize a project. This is probably a very subjective preference, but I feel less restricted since I'm not affected by editor capabilities, and specific ways of doing things.

For this reason I decided to go with DOME. It's very possible that if you decide to try out DOME, that you'll have to learn the programming language Wren, which shouldn't be hard to learn if you've used languages such as C/C++, C#, JavaScript and some other languages. Wren does have an active (but small) Discord community which can be found here: Wren Discord. There's a Discord community for DOME as well, which you can join here: DOME Discord.

I could have finished the development of Brainstuck twice during this entire process of finding the right engine. But, I now have a lot more experience with different engines and I now know what each of the engines are good for and what to use in certain scenarios. Which I hope will show in the future once I hopefully start releasing games.

Note: Godot is actually quite a good candidate for software development.
